# Foreman Vision — 2026-05-20

Captured from conversation with ej. This is the product vision, not the current state.

## What Foreman Actually Is

Foreman is a **company starter**. Not just a code verification tool.

You install it. It asks "What are you?" Based on your answer, it loads the right swarm template — the skills, inspectors, builders, and routing logic for that kind of work. Then it runs.

## The Onboarding Flow

```
$ foreman init

What kind of company are you building?
  1. Software / app development
  2. Creative writing / fiction / novels
  3. YouTube / video production
  4. Marketing / content agency
  5. Physical product / e-commerce
  6. Something else (describe it)

> [your answer]

✓ Loaded [domain] swarm template
✓ Inspector personas: [loaded from template]
✓ Builder: [auto-discovered from fleet]
✓ Foreman: arbitration + synthesis

Your company is ready. Run `foreman start` to begin.
```

## Swarm Templates (Skill Files)

Each domain gets a swarm template — a SKILL.md that defines:
- Builder patterns (what they implement)
- Inspector checklists (what they verify)
- Foreman arbitration rules (how decisions get made)
- Domain-specific routing (which model for which role)

| Domain | Builder | Inspectors | Foreman Rules |
|--------|---------|-----------|---------------|
| Software | Composer writes code | Opus checks bugs, regressions | Accept/reject/fix cycle |
| Creative writing | Composer writes chapters | VEGA (voice), RIFF (pacing), IRIS (character), FINCH (continuity), STORM (structure), ZIGGY (market) | Deluxe loop → revision plan |
| YouTube | Composer writes scripts + edits | Inspector checks pacing, hook, thumbnail | Accept/reject cycle |
| Marketing | Composer writes copy | Inspector checks brand alignment, channel fit | Accept/reject cycle |

Community members can write new templates. `foreman-skill-etsy-shop`, `foreman-skill-podcast`, etc.

## Architecture

```
You → Chat Interface (WebUI or terminal) → Foreman (orchestration brain)
                                              ├── Routes to the right swarm template
                                              ├── Dispatches builders
                                              ├── Dispatches inspectors
                                              ├── Arbitrates
                                              └── Optionally connects to Paperclip for visual dashboard
```

Foreman doesn't own the front-end. It's the brain in the middle that any interface can talk to.

## Paperclip Relationship

- Foreman standalone = headless (CLI-driven, chat-driven)
- Foreman + Paperclip = visual dashboard, org charts, Kanban boards, worktree management
- Paperclip is optional. A supercharger, not a dependency.
- NOT a plugin. Composes through API.

## StoryCraft Machine Integration

The New York Editors (VEGA, RIFF, IRIS, FINCH, STORM, ZIGGY) are already built as StoryCraft Machine editorial personas. They ARE Foreman inspectors with different checklists. The creative writing swarm template wraps StoryCraft's editorial room in Foreman's loop.

The pipeline: intake → dossier → draft → editorial → export is the same as:
builder writes → inspectors review in parallel → foreman synthesizes revision plan → builder revises → final verify

## Naming

- **Foreman** — current working name, works for construction/crew metaphor
- May feel too industrial for creative users. The product name and the engine name might differ.
- Template-level names (Editorial Room, Writing Room) can have their own personality while Foreman stays the infrastructure.
