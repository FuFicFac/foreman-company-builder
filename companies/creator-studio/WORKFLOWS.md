# WORKFLOWS.md -- Creator Studio

## Universal Intake

Every engagement begins with current-state discovery.

1. Ask where the content is today.
2. Inspect provided files/folders.
3. Identify missing core artifacts.
4. Infer the current stage.
5. Confirm the stage with the human.
6. Propose the next smallest useful workflow.

Never force a creator into a full pipeline if they only need the next stage.

## Stage Map

### Idea / Angle

Goal: turn the idea into a content project with a clear angle.

Outputs:

- angle summary;
- audience and platform target;
- working title/hook candidates;
- next research or drafting steps;
- workspace initialized.

### Notes / Research

Goal: organize source material into a usable content brain.

Outputs:

- research index;
- missing questions;
- source queue;
- outline options.

### Draft / Script

Goal: stabilize what exists and define the next content move.

Outputs:

- current content inventory;
- brand/voice notes;
- next sections or posts;
- revision vs drafting recommendation.

### Edit / Revision

Goal: execute bounded revision passes without losing the voice.

Outputs:

- revised content;
- change notes;
- unresolved questions;
- closeout report.

### Platform Package

Goal: prepare for publishing across platforms.

Outputs:

- platform-specific packages;
- captions and descriptions;
- tags and metadata;
- scheduling notes;
- final defects list.

### Publish / Launch

Goal: coordinate public release without making the creator become a project manager.

Outputs:

- launch checklist;
- announcement copy;
- cross-platform posts;
- audience follow-up plan;
- day-of launch heartbeat.

### Post-Publish / Analytics

Goal: learn from the release and keep the content alive.

Outputs:

- performance notes;
- audience feedback synthesis;
- follow-up content;
- relaunch opportunities;
- back-catalog maintenance tasks.

## First Tester Workflow

For early testers, use the smallest useful workflow:

```text
Intake -> Stage confirmation -> One focused deliverable -> Closeout report
```

Do not promise full automation. Prove usefulness with one piece of content, one stage, one visible result.
