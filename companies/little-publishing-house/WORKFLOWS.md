# WORKFLOWS.md — Little Publishing House

## Universal Intake

Every engagement begins with current-state discovery.

1. Ask where the book is today.
2. Inspect provided files/folders.
3. Identify missing core artifacts.
4. Infer the current stage.
5. Confirm the stage with the human.
6. Propose the next smallest useful workflow.

Never force a writer into a full pipeline if they only need the next stage.

## Stage Map

### Idea / Premise

Goal: turn the idea into a project workspace and a useful plan.

Outputs:

- premise summary;
- audience/genre/reader promise;
- working title candidates;
- next writing/research steps;
- workspace initialized.

### Notes / Dossier

Goal: organize source material into a usable book brain.

Outputs:

- dossier index;
- missing questions;
- research queue;
- outline options;
- living wiki seed.

### Outline

Goal: make the structure executable.

Outputs:

- chapter/section map;
- goals per chapter;
- constraints/non-goals;
- draft order;
- risk list.

### Partial Draft

Goal: stabilize what exists and define the next writing move.

Outputs:

- current manuscript inventory;
- continuity/style notes;
- next chapters/scenes/sections;
- revision vs drafting recommendation.

### Complete Draft

Goal: move from manuscript to editorial plan.

Outputs:

- editorial diagnosis;
- revision map;
- priority fixes;
- specialist reports if needed;
- human decision on revision scope.

### Revision / Edit

Goal: execute bounded revision passes without losing the voice.

Outputs:

- revised sections;
- change notes;
- unresolved questions;
- continuity updates;
- closeout report.

### Proof / Production

Goal: prepare for export and distribution.

Outputs:

- proof checklist;
- export requirements;
- EPUB/PDF/print package status;
- ISBN/edition notes if relevant;
- final defects list.

### Metadata / Storefront

Goal: make the book legible and sellable.

Outputs:

- title/subtitle options;
- categories/keywords;
- product description;
- author bio;
- comp/positioning notes;
- reader-facing promise.

### Launch

Goal: coordinate public release without making the writer become a project manager.

Outputs:

- launch checklist;
- announcement copy;
- newsletter sequence;
- social posts;
- review/ARC plan;
- day-of launch heartbeat.

### Post-Launch / Relaunch

Goal: learn from the release and keep the book alive.

Outputs:

- performance notes;
- reader feedback synthesis;
- follow-up content;
- relaunch opportunities;
- backlist maintenance tasks.

## First Patreon Tester Workflow

For early testers, use the smallest useful workflow:

```text
Intake → Stage confirmation → One focused deliverable → Closeout report
```

Do not promise full automation. Prove usefulness with one book, one stage, one visible result.
